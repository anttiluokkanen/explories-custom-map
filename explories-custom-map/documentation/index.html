    Getting started

    1. Include JavaScript files

    Include jQuery, ECM js and Google Maps in anywhere in the head or body of your HTML document:

    

        
    jQuery: ECM requires jQuery. If your site uses jQuery already please include it before the ECM js file. If you're site doesn't use jQuery already you may inlcude it from the ECM package vendor directory or jQuery CDN.
    

        
    ECM JavaScript: The ECM script element must have id attribute with value ecmScript.
    

        
    Google Maps: ECM requires Google Maps API. The Google Maps API script element must have the callback parameter with value ECM.init. ECM requires the following Google APIs enabled for your API key:
    

        
      
            
    • Maps JavaScript API
      • 
            
    • Directions API (only if using routes with travelMode enabled)
      • 
        
    

        
    

        
    2. Include ECM styleheet
    

        
    Include ECM stylesheet in the head of your HTML document:
    

        
    

        
    This is the common stylesheet for ECM. All theme specific styles are in a separate theme stylesheet which is loaded automatically (please see chapter Theme styles for more information).
    

        
    3. Add markup
    

        
    Add a div element for ECM in your HTML document body. The div element must have an id attribute with value ecm.
    

        
    

    
    


    
    

        
    Configuration
    

        
    Configuration is set with div element data attributes and theme configuration file. Depending on the system you use, these settings may be configurable in your CMS (for example WordPress) dashboard.
    

        
    Div element data attributes
    

        
    

        
    
            
                
    
                    Attribute
                    Description
                
    
                
            
                
    
                    data-theme
                    Local theme directory name if using a custom theme.
                
    
                
    
                    data-theme-key
                    External theme key. This is used to load the theme from explories.net server.
                
    
                
    
                    data-lan
                    ECM language.
                
    
                
    
                    data-offset-top
                    ECM offset from the top of document body. If for example the site header height is 80px the data-offset-top value should be "80px".
                
    
                
    
                    data-feed
                    If given with any value, for example data-feed="true", ECM will initialize with feed mode on.
                
    
                
    
                    data-article
                    The id of an Explores article which is opened to ECM internal article view on init.
                
    
                
        
    

        
    Theme configuration file
    

        
    Default configuration can be overriden in theme configuration file located in themes/themename/config.js. The configuration file consists of two function calls: setConfig and setTexts. To these are passed options objects. Properties defined in the options objects will override default values.

    Markers
    

        
    Markers are defined in an array of marker objects in json format. The example below shows a marker object with all possible properties. Marker properties are listed below.
    

        
    

        
    Marker properties
    

        
    
            
                
    
                    Property
                    Type
                    Description
                
    
                
            
                
    
                    id
                    number
                    Marker unique identifier.
                
    
                
    
                    title
                    string
                    Marker title which is displayed as a heading in the marker info window.
                
    
                
    
                    image
                    string
                    Absolute or relative path to an image file to be shown in the marker info window.
                
    
                
    
                    latitude
                    number
                    Marker latitude. Latitude ranges between -90 and 90 degrees, inclusive. Values above or below this range will be clamped to the range [-90, 90]. This means that if the value specified is less than -90, it will be set to -90. And if the value is greater than 90, it will be set to 90.
                
    
                
    
                    longitude
                    number
                    Marker longitude. Longitude ranges between -180 and 180 degrees, inclusive. Values above or below this range will be wrapped so that they fall within the range. For example, a value of -190 will be converted to 170. A value of 190 will be converted to -170. This reflects the fact that longitudes wrap around the globe.
                
    
                
    
                    url
                    string
                    Absolute or relative url which is opened by clicking the marker info window content.
                
    
                
    
                    target
                    string
                    The target for the link defined by property url. Use "_blank" to open the link in a new browser window/tab.
                
    
                
    
                    icon
                    string
                    Absolute or relative path to an image file to be used as the marker icon.
                
    
                
    
                    modes
                    array
                    Array of map mode name strings to define the map modes in which to show this marker. If modes array is not defined this marker will be shown in all map modes.
                
    
                
    
                    tags
                    array
                    Array of tag objects associated with this marker. Tag icons are shown in the marker info window. If any filters are set this marker will be shown only if filter settings match the tags.

    External markers
    

        
    Map markers loaded from external sources are defined in the External markers object in the theme configuration file. The example below defines one set of external markers to be loaded via Explories API.
    

        
    externalMarkersSets: TO DO
    

        
    Each external marker set is defined with unique property name. The marker set label can be defined in the configuration texts options with the corresponding property name.
    

        
    

        
    Parameter values {lat} and {lng} are replaced by config.initLocation.lat and config.initLocation.lng automatically.
    

        
    Marker set object properties
    

        
    
            
                
    
                    Property
                    Type
                    Description
                
    
                
            
                
    
                    url
                    string
                    Url from where the markers data is loaded.
                
    
                
    
                    iconW
                    number
                    Marker icon width in pixels.
                
    
                
    
                    iconH
                    number
                    Marker icon height in pixels.
                
    
                
    
                    checked
                    boolean
                    Wheater to load these markers on init or not.
                
    
                
    
                    reloadOnBoundsChange
                    boolean
                    Wheater to reload these markers on Google Maps bound change or not.
                
    
                
    
                    callbackName
                    string
                    Name of the callback function to be called after the markers have been loaded. This is used to convert external data to a format understandable to ECM. Each different source must have it's own conversion function coded in ECM.
                
    
                
    
                    articleViewUrl
                    string
                    If external markers are Explories articles they can be opened in ECM internal article view. This articleViewUrl defines the url to the article view configured ECM page. Url can have parameters id and slug defined with curly braces, for example https://path/to/map/{id}/{slug}.
                
    
                
    
                    external
                    boolean
                    Wheater to mark these markers with style external or not.
                
    
                
        
    

    
    



    
    

        
    Map layers
    

        
    Map layers loaded from external sources are defined in the mapLayers array in the theme configuration file. The example below defines a map layer object using OpenTopoMap API.
    

        
    

        
    Map layer properties
    

        
    
            
                
    
                    Property
                    Type
                    Description
                
    
                
            
                
    
                    name
                    string
                    Map layer unique name. Name must be a valid JavaScript identifier, i.e. a sequence of alphanumerical characters, also including the underscore ("_") and dollar sign ("$"), that cannot start with a number.
                
    
                
    
                    type
                    string
                    Possible values are "imageMapType" and "kml". See Google Maps API documentation for more information about Image Map Type and KML Layers
                
    
                
    
                    url
                    string
                    Map layer API url where to load the tiles. Parameters zoom, x and y in the url are marked with brackets.
                
    
                
    
                    minZoom
                    number
                    Minimum Google Maps zoom level to use this map layer.
                
    
                
    
                    maxZoom
                    number
                    Maximum Google Maps zoom level to use this map layer.
                
    
                
    
                    bounds
                    array
                    Bounds for this map layer. If the Google Maps view if out of these bounds this map layer is not used. Bounds array format is [SouthWestLng, SouthWestLat, NortEastLng, NorthEastLat].
                
    
                
    
                    checked
                    boolean
                    Defines if this map layer is on by default.
